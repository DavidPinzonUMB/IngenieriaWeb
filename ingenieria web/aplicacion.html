Problemática

Cuando varias personas comparten gastos como en el caso de roomies, viajes familiares, grupos de amigos o parejas, suele haber confusión sobre cuánto debe pagar cada uno.

Los problemas más comunes son errores al dividir los gastos, que alguien consuma más que otros, falta de control financiero y conflictos por dinero.

Solución Propuesta

Se propone desarrollar una aplicación web cliente que permita registrar gastos compartidos y calcular automáticamente cuánto le corresponde pagar a cada persona.

La aplicación permitirá:

  • Crear grupos de gastos.
  • Registrar pagos realizados.
  • Dividir gastos por categorías (viaje, servicios públicos, otros).
  • Establecer metas de ahorro.
  • Visualizar automáticamente quién debe y cuánto debe.
  • Actualizar balances cuando se abonen o paguen gastos.
  • Realizar pagos automáticos autorizados.

La aplicación funcionará como cliente enviando y recibiendo datos desde un servidor local, cumpliendo el modelo Cliente-Servidor.

Arquitectura del Sistema

Cliente

Se encargará del registro de usuarios, creación de grupos, registro de gastos y visualización de resultados mediante una interfaz amigable.

Servidor (Local)

Recibirá las solicitudes del cliente, validará los datos y realizará las consultas necesarias.

Base de Datos

Se utilizará el gestor MariaDB para realizar operaciones CRUD (Crear, Leer, Actualizar y Eliminar registros).
